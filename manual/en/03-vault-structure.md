## Organization philosophy

**Simplicity first:**
- Minimal folder structure
- Semantic organization via **tags** (not folders)
- The AI agent "knows" how to work thanks to the files in the `system/` folder ‚Äì all of them are loaded automatically at every session start
- Every note type has a **template** ‚Üí consistency without effort

**Language of system/ and templates/:** Files are in English ‚Äì Claude responds in the language you write in (write in Slovak ‚Üí get Slovak responses). Template sections (Overview, Goals...) are in English, but you write note content in any language.

## Tag system

Tags are the **primary organization method** ‚Äì more important than folders.

### Required tags (every note must have at least one)

**Note type (`#type/`):**

| Tag | Use |
|---|---|
| `#type/project` | Project note, project overview |
| `#type/task` | Task, to-do item |
| `#type/note` | Free-form note, idea |
| `#type/reference` | Documentation, tutorial, guide |
| `#type/conversation` | AI conversation record |
| `#type/decision` | Decision record (ADR) |

**Status (`#status/`):**

| Tag | Use |
|---|---|
| `#status/draft` | Work in progress |
| `#status/active` | Actively being worked on |
| `#status/done` | Completed |
| `#status/archived` | Archived |

### Recommended tags

**Project:** `#project/[project-name]`

**Topic:** `#topic/ai`, `#topic/development`, `#topic/obsidian`, `#topic/infrastructure`, ...

**Priority (for tasks):** `#priority/high`, `#priority/medium`, `#priority/low`

## Frontmatter standard

Every note should have a YAML frontmatter:

```yaml

  - type/project
```

## Installing the vault structure

### Option A: GitHub Release (recommended)

Download the ZIP from GitHub releases:

```
https://github.com/[username]/opencode-obsidian-ai-workspace/releases
```

Extract to the vault root:

```bash
unzip opencode-obsidian-ai-workspace.vX.Y.Z.zip -d /PATH/TO/YOUR/VAULT/
```

The ZIP contains:
- `system/` ‚Üí extracts to `vault/system/` (EN rules for the agent)
- `templates/` ‚Üí extracts to `vault/templates/` (EN sections, write content in any language)
- `opencode-obsidian-ai-workspace.vX.Y.Z/` ‚Üí manuals

### Option B: Clone from GitHub repo

```bash
git clone https://github.com/[username]/opencode-obsidian-ai-workspace.git /tmp/workspace-repo

cp -r /tmp/workspace-repo/system/* /PATH/TO/YOUR/VAULT/system/
cp -r /tmp/workspace-repo/templates/* /PATH/TO/YOUR/VAULT/templates/

rm -rf /tmp/workspace-repo
```

## Key files ‚Äì overview

### `system/opencode-chat-settings.md`

**Auto-generated file** ‚Äì the OpenCode Chat plugin writes (and updates) this file automatically whenever settings change. It contains the current configuration values:

```markdown
- **Projects folder:** `projects`
- **Export folder:** `conversations`
- **Rules path:** `system`
```

The agent reads these values at the start of every session to know where to store notes. **Do not edit this file manually** ‚Äì make changes via **Settings ‚Üí OpenCode Chat** in Obsidian.

### `system/opencode-rules.md`

**The main rules file.** Defines rules for the AI agent:
- Folder structure and where to store notes
- Tag hierarchy (required tags, recommended)
- Frontmatter standard
- How to use templates
- File naming (lowercase, hyphens)
- Workflow: creation, update, archiving

Files in `system/` are in English ‚Äì Claude understands them and responds in your language. All files in the `system/` folder are loaded automatically at every session start ‚Äì no need to send them manually.

> **The projects folder** is not hardcoded in the rules ‚Äì the agent always reads the current value from `system/opencode-chat-settings.md`, where the plugin keeps it up to date.

### `system/quick-guide.md`

A concise guide **for you** ‚Äì how to search, create notes, workflow.

### `system/tag-index.md`

Complete list of all tags in your vault with descriptions. Add new tags here as you create them.

### `templates/`

Templates for every note type. Sections are in English (Overview, Goals, Description...) ‚Äì you write the content in any language. The agent uses them automatically when creating new notes.

## Verifying the installation

After installation, verify in OpenCode Chat:

```
Summarize the note organization rules for this vault
```

The AI should summarize the rules ‚Äì all files in `system/` are loaded automatically.

Then test note creation:

```
Create a project note for project "test-workspace"
```

The AI should:
1. Load `templates/project.md`
2. Fill in the placeholders
3. Create the file in the configured projects folder (e.g. `projects/test-workspace/overview.md`)
4. Add correct tags and frontmatter

**‚úÖ Checkpoint:** Vault structure works, agent uses rules and templates

## Customizing for your workflow

### Add your own rules

Edit `system/opencode-rules.md` ‚Äì add a section:

```markdown
## Custom rules

### Note language
- Primary language: English

### Projects
- Each project has subdirectories: docs/, tasks/
- Archived projects: archive/YYYY/[project]/

### Tasks
- Deadline in frontmatter: `due: YYYY-MM-DD`
```

### Create a custom template

Copy an existing template and modify:

```
AI: Copy templates/note.md to templates/meeting.md and add sections: Attendees, Agenda, Action Items
```
