# OpenCode Obsidian AI Workspace – Vault Structure

This document describes the vault organization system and how to install it.

**Related docs:**
- [[02-bedrock-setup]] – Previous step
- [[04-workflow]] – Next step

---

## Organization philosophy

**Simplicity first:**
- Minimal folder structure
- Semantic organization via **tags** (not folders)
- The AI agent "knows" how to work thanks to the files in the `x-ai-rules/` folder – all of them are loaded automatically at every session start
- Every note type has a **template** → consistency without effort

**Language of x-ai-rules/ and x-ai-templates/:** Files are in English – Claude responds in the language you write in (write in Slovak → get Slovak responses). Template sections (Overview, Goals...) are in English, but you write note content in any language.

---

## Folder structure

```
vault/
├── x-ai-rules/            ← rules and guides for the agent + you
│   ├── opencode-rules.md         ← main rules for the OpenCode agent
│   ├── opencode-chat-settings.md ← current plugin settings (auto-generated)
│   ├── quick-guide.md            ← quick reference for you
│   └── tag-index.md              ← all tags and their descriptions
├── x-ai-templates/        ← templates for every note type
│   ├── project.md
│   ├── task.md
│   ├── note.md
│   ├── reference.md
│   ├── decision.md
│   └── conversation.md
├── [projects-folder]/     ← active projects – folder is configurable in opencode-chat (default: projects/)
│   └── [project-name]/
├── archive/               ← archived notes and projects
└── [export-folder]/       ← AI conversation records (default: conversations/)
```

---

## Tag system

Tags are the **primary organization method** – more important than folders.

### Required tags (every note must have at least one)

**Note type (`#type/`):**

| Tag | Use |
|---|---|
| `#type/project` | Project note, project overview |
| `#type/task` | Task, to-do item |
| `#type/note` | Free-form note, idea |
| `#type/reference` | Documentation, tutorial, guide |
| `#type/conversation` | AI conversation record |
| `#type/decision` | Decision record (ADR) |

**Status (`#status/`):**

| Tag | Use |
|---|---|
| `#status/draft` | Work in progress |
| `#status/active` | Actively being worked on |
| `#status/done` | Completed |
| `#status/archived` | Archived |

### Recommended tags

**Project:** `#project/[project-name]`

**Topic:** `#topic/ai`, `#topic/development`, `#topic/obsidian`, `#topic/infrastructure`, ...

**Priority (for tasks):** `#priority/high`, `#priority/medium`, `#priority/low`

---

## Frontmatter standard

Every note should have a YAML frontmatter:

```yaml
---
created: 2026-03-01
updated: 2026-03-01
tags:
  - type/project
  - status/active
  - project/my-project
  - topic/development
---
```

## Installing the vault structure

### Option A: GitHub Release (recommended)

Download the ZIP from GitHub releases:

```
https://github.com/[username]/opencode-obsidian-ai-workspace/releases
```

Extract to the vault root:

```bash
unzip opencode-obsidian-ai-workspace.vX.Y.Z.zip -d /PATH/TO/YOUR/VAULT/
```

The ZIP contains:
- `x-ai-rules/` → extracts to `vault/x-ai-rules/` (EN rules for the agent)
- `x-ai-templates/` → extracts to `vault/x-ai-templates/` (EN sections, write content in any language)
- `opencode-obsidian-ai-workspace.vX.Y.Z/` → manuals

### Option B: Clone from GitHub repo

```bash
git clone https://github.com/[username]/opencode-obsidian-ai-workspace.git /tmp/workspace-repo

cp -r /tmp/workspace-repo/x-ai-rules/* /PATH/TO/YOUR/VAULT/x-ai-rules/
cp -r /tmp/workspace-repo/x-ai-templates/* /PATH/TO/YOUR/VAULT/x-ai-templates/

rm -rf /tmp/workspace-repo
```

## Key files – overview

### `x-ai-rules/opencode-chat-settings.md`

**Auto-generated file** – the OpenCode Chat plugin writes (and updates) this file automatically whenever settings change. It contains the current configuration values:

```markdown
- **Projects folder:** `projects`
- **Export folder:** `conversations`
- **Rules path:** `x-ai-rules`
```

The agent reads these values at the start of every session to know where to store notes. **Do not edit this file manually** – make changes via **Settings → OpenCode Chat** in Obsidian.

### `x-ai-rules/opencode-rules.md`

**The main rules file.** Defines rules for the AI agent:
- Folder structure and where to store notes
- Tag hierarchy (required tags, recommended)
- Frontmatter standard
- How to use templates
- File naming (lowercase, hyphens)
- Workflow: creation, update, archiving

Files in `x-ai-rules/` are in English – Claude understands them and responds in your language. All files in the `x-ai-rules/` folder are loaded automatically at every session start – no need to send them manually.

> **The projects folder** is not hardcoded in the rules – the agent always reads the current value from `x-ai-rules/opencode-chat-settings.md`, where the plugin keeps it up to date.

### `x-ai-rules/quick-guide.md`

A concise guide **for you** – how to search, create notes, workflow.

### `x-ai-rules/tag-index.md`

Complete list of all tags in your vault with descriptions. Add new tags here as you create them.

### `x-ai-templates/`

Templates for every note type. Sections are in English (Overview, Goals, Description...) – you write the content in any language. The agent uses them automatically when creating new notes.

## Verifying the installation

After installation, verify in OpenCode Chat:

```
Summarize the note organization rules for this vault
```

The AI should summarize the rules – all files in `x-ai-rules/` are loaded automatically.

Then test note creation:

```
Create a project note for project "test-workspace"
```

The AI should:
1. Load `x-ai-templates/project.md`
2. Fill in the placeholders
3. Create the file in the configured projects folder (e.g. `projects/test-workspace/overview.md`)
4. Add correct tags and frontmatter

**✅ Checkpoint:** Vault structure works, agent uses rules and templates

## Customizing for your workflow

### Add your own rules

Edit `x-ai-rules/opencode-rules.md` – add a section:

```markdown
## Custom rules

### Note language
- Primary language: English

### Projects
- Each project has subdirectories: docs/, tasks/
- Archived projects: archive/YYYY/[project]/

### Tasks
- Deadline in frontmatter: `due: YYYY-MM-DD`
```

### Create a custom template

Copy an existing template and modify:

```
AI: Copy x-ai-templates/note.md to x-ai-templates/meeting.md and add sections: Attendees, Agenda, Action Items
```
